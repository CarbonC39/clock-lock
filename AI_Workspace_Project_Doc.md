# Clock Lock

## 1. 项目概述
**Clock Lock** 是一个专为个人开发者（特别是 ADHD 群体，同时也适用于神经典型人群）设计的辅助开发环境。 其核心理念是提供一个“赛博工友”，通过技术支持、情感陪伴、进度督促和知识管理，降低启动阻力，缓解开发焦虑。

## 2. 技术栈
* **Tauri+Vue.js+Rust**
*  **数据存储**：
  * 项目状态：基于本地目录树结构与隐藏目录下的数个 Markdown 文件。
  * 记忆存储：基于 SQLite 的全文搜索扩展，存储历史对话与事件日志。对话支持记忆压缩。


## 3. 核心架构与功能设计

### 3.1 知识库与内存管理
* 支持挂载本地文件夹和云端文件夹作为项目知识来源。
* **混合追踪机制:**
    * **Git 追踪:** 利用 Rust 的 `git2` 库监听代码变动。
    * **文件系统监听** 利用 `notify` crate 监听非代码文件（二进制、素材、文档）的变动。
    * 不追踪`.gitignore`里的文件。
* **记忆存储:**采用“Markdown-as-Database”策略，在应用数据目录为每个项目创建文件夹。以及sqlite。
* **二进制文件处理:** 用户可以选择为二进制文件（如模型、素材）添加“注释”，存入本地元数据，这样agent就可以追踪二进制文件进度。但是拒绝强制填写或者催促填写。

### 3.2 交互模式
* 左侧文件目录，中间首页 Markdown，右侧 Agent 聊天窗口。样式参考vscode，但是不要做得像重型ide，保持扁平简介。推荐的用色是饱和度低的白、蓝、粉、紫。支持语法高亮和Diff展示。动画要丝滑，不要使用太多hover特效。支持明亮/黑暗模式。UI现在都用英语。
* **首页 Markdown:** 作为项目的“存档点”，解决 ADHD 用户的上下文遗忘问题。包含包含 Overview, Progress, Todos等等，用户可以和agent一起编辑。
* **小组件模式:** 支持缩小为桌面悬浮窗，可切换显示当前唯一 Todo 和极简 Agent 状态。

### 3.3 Agent 能力与执行权限
* **Bash 执行:** Agent 不是coding agent，它只可生成“推荐操作”。
* **受控沙箱:**
    * 所有编辑命令需用户点击“Approve & Run”后执行，只有只读命令可以自动进行。
    * 执行目录强制锁定在 Workspace 目录。
    * 内置高危命令黑名单。
* **协作模式:** 采用“Draft & Review”模式，Agent 生成修改建议或草稿文件，由用户确认合并，避免直接覆盖代码。
* agent支持API和本地，采用openai兼容格式。支持流式输出。

## 4. 督工与情绪价值设计

### 4.1 督促机制 
- 基于设置配置（如：程序挂机超过 48 小时未更新）触发系统请求agent，生成通知。用户拥有“叫停权”和“勿扰模式”，规避需求回避心理。用户也可以通过自然语言让agent推延下次监督的时间，比如“我下周要考试”。

- 通过小组件展示 Agent 的实时状态（如：“Agent 正在阅读文档...”），营造陪伴感。

### 4.2 形象与表达
* 采用拓麻歌子式的像素颜文字状态机。
* 用户可自定义agent性格。
* **沟通准则:** 针对错误而非针对用户进行描述，缓解拒绝敏感。在技术反馈中加入适度的夸夸或心理支持。
